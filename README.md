# 高性能分布式流媒体缓存系统

## Overview
一个专为 HLS/DASH 视频切片（如 `.ts` 文件）设计的智能缓存系统，支持多级缓存、自适应预取、热点加速与穿透防护。

## 目前项目实现的功能

- 使用 Go 语言编写，具有高并发和高性能的特点。
- 使用一致性哈希算法实现了节点选择，保证了缓存的负载均衡。
- 使用 LRU 算法结合 TTL（生存时间）实现了带过期机制的缓存淘汰策略，在保障内容新鲜度的同时最大化命中率。
- 基于 gRPC 实现高效的节点间通信：缓存未命中时，通过 gRPC 协议向负责该 key 的 peer 节点发起远程获取请求，序列化效率高、延迟低。
- 使用 protobuf 实现了节点间的二进制通信，提高了传输的效率和兼容性。
- 集成 etcd 实现服务注册与发现：各缓存节点启动时自动注册到 etcd 集群，支持弹性扩缩容。
- 集成 Redis 作为分布式共享缓存层，跨节点复用热点数据，进一步减少回源压力。
- 实现智能预取机制，根据用户播放行为（连续访问模式）与实时网络带宽，动态预加载后续视频切片。
- 冷启动时自动预热视频片头，降低首次访问延迟。
- 运用空值缓存策略，对不存在的文件路径缓存短 TTL 空结果，有效抵御缓存穿透攻击。
- 使用 singleflight 实现了防止缓存击穿的机制，避免了缓存失效时的并发风险。
